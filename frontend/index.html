Features

What I Do

HTML

HTML로 다양한 레이아웃을 배치할 수 있습니다. 특히 flex와 grid 형식의 레이아웃을 적극적으로 사용하려고 노력하고 있습니다.

CSS

자주 사용되는 css 태그, 속성에 대해 이해하고 있습니다. 사용자 경험 향상을 위해 반응형 레이아웃에 관한 연습을 하고 있습니다.

JavaScript

기본적인 문법과 간단한 DOM 조작에 대해 숙지하고 있습니다. JS의 이론적인 내용에 약해 유튜브나 블로그 등에서 이론에 대한 아티클들을 자주 읽으려 노력합니다.

ReactJS

useEffect, useState의 작동 원리를 간단히 알고 있고, API로부터 데이터를 받아와 바인딩시킬 수 있습니다. 역시 이론적인 내용에 약하다고 생각해 다양한 아티클들을 자주 읽으려 노력합니다.

TailwindCSS

대표적인 Utility-First CSS로, css파일과 html 파일을 왔다갔다 할 필요 없이 인라인으로 빠르게 컴포넌트들을 완성할 수 있는 라이브러리입니다. 개인적으로 가장 선호하는 css라이브러리입니다.

Styled Components

React에서 사용 가능한 CSS-in-JS 라이브러리입니다. 컴포넌트별로 스타일을 지정해두고 재사용할 일이 많을 때 효율이 극대화됩니다.

Github

가장 많이 사용하는 원격 소스코드 저장소입니다. git 및 github를 사용할 때 필요한 대부분의 명령어를 숙지하고 있습니다. 협업 컨벤션에 관심이 많습니다.

FastAPI

CRUD API를 쉽고 빠르게 개발할 수 있고, API 문서화를 자동으로 수행해주는 라이브러리입니다. GET, POST 메소드에 해당하는 API를 만들어본 경험이 있고, 필요에 따라 추가적인 공부를 진행하고 있습니다.

Firebase

Google에서 개발한 백엔드 서비스입니다. Firestore와 Firebase Cloud Message(FCM)을 깊게 사용해본 경험이 있습니다.

CollaboraMate | 개발기간: 24.01.14 ~ 24.02.25 | 개발 인원: 2명

조별 과제 멤버를 편하게 구하는 앱을 개발했습니다.

앱 개발은 크로스 플랫폼 개발이 가능한 React Native CLI를 사용했습니다.

데이터베이스로는 Firestore, Realtime Database를 사용했습니다.
Firestore에는 유저 정보, 게시물 정보를 저장했고,
Realtime Database에는 조별 과제 팀이 구성되면 자동으로 생성되는 채팅방의 채팅 내용을 저장했습니다.

사용자가 작성한 멤버 모집 글에 누군가가 지원하거나, 사용자가 지원한 팀에서 함께 하자는 제안이 들어오면, FCM과 Notifee를 활용해 Push 알림을 전송하도록 구현했습니다.

인공지능융합대학으로 사용 범위를 한정하는 대신, Github와 기술 스택을 프로필에 반드시 명시하도록 하고, 팀원을 구하거나 팀을 구할 때 Github 및 기술 스택을 확인할 수 있도록 했습니다.

처음으로 진지하게 서비스 개발을 임하고, 배포까지 해 봤던 프로젝트라 많은 애정이 있는 앱이지만, 시간의 여유가 없고 해 보고 싶은 다른 개발이 너무 많아 유지 보수는 하지 못하고 있어 아쉬운 마음이 남아있습니다.

Genshin.gg | 개발기간: 24-03-15 ~ 현재 | 개발 인원: 1명

예전에 즐겨하던 "원신" 이라는 게임의 계정ID를 입력하면, 해당 계정의 다양한 정보를 보여주는 웹사이트를 만들어보는 개인 프로젝트입니다.

웹 개발은 React를 사용했고, Web Application Server는 FastAPI로 구축했습니다.
그리고 get요청과 post 요청을 편하게 수행하기 위해 axios 라이브러리를 사용했습니다.

이미 유사한 서비스를 제공하는 "Enka Network"에서 공개 API를 제공하고 있기 때문에, 이 API를 활용해 개발했습니다.

CORS(Cross-Origin Resource Sharing) 문제를 해결하기 위해 FastAPI로 백엔드를 구축하고, Client가 FastAPI로 get 요청을 하면, FastAPI가 Enka Network API로 get 요청을 하고, 결과를 받아와 데이터를 가공한 뒤, Client에게 전달해주도록 구현해서 해결했습니다.

TailwindCSS를 도입해 공부하며 레이아웃을 구성하고 있습니다. 이미 완성된 디자인대로 만드는 상황이 아니라, 다양한 레이아웃을 배치해보며 디자인하는 상황인 경우 TailwindCSS의 장점이 잘 드러나는 것 같습니다.

반응형 레이아웃을 도입해 모바일, 태블릿, PC 모든 환경에서 최적의 경험을 할 수 있도록 여러 레이아웃을 실험해보고 있습니다.
