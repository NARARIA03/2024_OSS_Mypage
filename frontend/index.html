Features

What I Do

HTML

HTML로 다양한 레이아웃을 배치할 수 있습니다. 특히 flex와 grid 형식의 레이아웃을 자주 사용합니다.

CSS

자주 사용되는 css 태그, 속성에 대해 이해하고 있습니다. 사용자 경험 향상을 위해 반응형 레이아웃에 관한 연습을 하고 있습니다.

JavaScript

기본적인 문법과 간단한 DOM 조작에 대해 숙지하고 있습니다. JS의 이론적인 내용에 약하다고 생각해 공식문서와 블로그 글 등을 통해 공부중입니다.

ReactJS

React Hook들의 작동 결과를 알고 있고, 컴포넌트 단위 개발에 익숙합니다. React 역시 이론적인 내용에 약하다고 생각해 공식문서와 블로그 글 등을 통해 공부중입니다.

TailwindCSS

대표적인 Utility-First CSS로, 축약된 이름의 class를 추가하는 것만으로 css속성을 적용할 수 있습니다. 개인적으로 매우 선호하는 라이브러리입니다.

Styled Components

React에서 사용 가능한 CSS-in-JS 라이브러리입니다. 컴포넌트별로 스타일을 지정해 사용하는 방식으로 CSS 오염을 방지합니다.

Github

git 및 github를 사용할 때 중요한 명령어를 숙지하고 있습니다. 협업 컨벤션에 관심을 가지고 좋은 커밋 메시지를 작성하려 노력합니다.

FastAPI

백엔드 API를 쉽고 빠르게 개발할 수 있고, API 문서화를 자동으로 수행해주는 장점이 있습니다. 최근 계속 사용하고 있는 Python 웹 프레임워크입니다.

Firebase

Google에서 개발한 애플리케이션 개발 플랫폼입니다. Firestore(DB)와 Firebase Cloud Message(푸시 알림) 기능을 깊게 사용해본 경험이 있습니다.

CollaboraMate | 개발기간: 24.01.14 ~ 24.02.25 | 개발 인원: 2명

조별 과제 팀원을 편하게 구하는 앱을 개발했습니다.

앱 개발은 크로스 플랫폼 개발이 가능한 React Native CLI를 사용했습니다.

데이터베이스로는 Firestore, Realtime Database를 사용했습니다.
Firestore에는 유저 정보, 게시물 정보를 저장했고,
Realtime Database에는 조별 과제 팀이 구성되면 자동으로 생성되는 채팅방의 채팅 내용을 저장했습니다.

사용자가 작성한 멤버 모집 글에 누군가가 지원하거나, 사용자가 지원한 팀에서 함께 하자는 제안이 들어오면, FCM과 Notifee를 활용해 Push 알림을 전송하도록 구현했습니다.

인공지능융합대학으로 사용 범위를 한정하는 대신, Github와 기술 스택을 프로필에 반드시 명시하도록 하고, 팀원을 구하거나 팀을 구할 때 Github 및 기술 스택을 확인할 수 있도록 했습니다.

처음으로 진지하게 서비스 개발을 임하고, 배포까지 해 봤던 프로젝트라 많은 애정이 있는 앱이지만, 시간의 여유가 없고 해 보고 싶은 다른 개발이 너무 많아 유지 보수는 하지 못하고 있어 아쉬운 마음이 남아있습니다.

Genshin.gg | 개발기간: 24-03-15 ~ 진행중 | 개발 인원: 1명

예전에 즐겨하던 "원신" 이라는 게임의 계정ID를 입력하면, 해당 계정의 다양한 정보를 보여주는 웹사이트를 만들어보는 개인 프로젝트입니다.

FE는 ReactJS를 사용했고, Web Application Server는 FastAPI와 uvicorn으로 구축했습니다.
또한 get요청과 post 요청을 편하게 수행하기 위해 axios 라이브러리를 사용했습니다.

이미 유사한 서비스를 제공하는 "Enka Network"에서 공개 API를 제공하고 있기 때문에, 이 API를 활용해 개발했습니다.

Enka Network API로 FE에서 axios를 사용해 바로 요청하자 CORS(Cross-Origin Resource Sharing) 문제가 발생해, 이를 해결하기 위해 FastAPI로 BE를 구축하고, CORSMiddleware와 httpx 라이브러리를 사용해 Client가 FastAPI로 get 요청을 하면, FastAPI가 Enka Network API로 get 요청을 하고, 결과를 받아와 데이터를 가공한 뒤, Client에게 전달해주도록 구현해서 해결했습니다.

TailwindCSS를 도입해 공부하며 레이아웃을 구성하고 있습니다. 이미 완성된 디자인대로 만드는 상황이 아니라, 다양한 레이아웃을 배치해보며 디자인하는 상황일 때 TailwindCSS의 장점이 잘 드러나는 것 같습니다.

반응형 레이아웃을 도입해 모바일, 태블릿, PC 모든 환경에서 최적의 경험을 할 수 있도록 여러 레이아웃을 실험해보고 있습니다.
